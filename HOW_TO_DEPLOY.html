Deployment Guide v2 — with Shared Database

🏫 NAFL PTM Portal — Full Setup

Free forever · Real-time shared data · Works on all devices simultaneously

✅ This version uses Google Sheets as a shared database. Admin uploads data once → every teacher on every device sees the same live data immediately.
What you'll set up (in order):
  1. Part 1 — Create the Google Sheets backend (10 min)
  2. Part 2 — Connect the portal to your backend (2 min)
  3. Part 3 — Deploy the portal to GitHub Pages (10 min)

📊 Part 1 — Create the Google Sheets Backend

Step 1 — Create a new Google Sheet
Go to sheets.new (opens a blank sheet directly)
Rename it to NAFL PTM Database (click the title at top-left)
Step 2 — Open the Script Editor
Click Extensions (top menu) → Apps Script. A new tab opens with a code editor.
Step 3 — Paste the backend code
1. Select ALL existing code (Ctrl+A / Cmd+A) and delete it
2. Open GoogleAppsScript.gs from this download, copy its entire content
3. Paste into the Apps Script editor
4. Click the 💾 Save icon (or Ctrl+S)
Step 4 — Initialize the sheets
1. In the function dropdown (top toolbar), select initializeSheets
2. Click ▶ Run
3. Click Review permissions → choose your account → Advanced → Go to NAFL PTM Database (unsafe) → Allow
⚠️ The "unsafe" warning is normal Google wording for self-written scripts. It only accesses your own sheet.
Step 5 — Deploy as a Web App
1. Click Deploy (top right) → New deployment
2. Click ⚙️ next to "Select type" → Web app
3. Description: NAFL PTM API v1
4. Execute as: Me  ·  Who has access: Anyone
5. Click Deploy → Authorize again if prompted
Step 6 — Copy your Web App URL
A box shows your Web app URL:
https://script.google.com/macros/s/AKfycb................................/exec
Copy this entire URL — needed in Part 2.
✅ Test it: paste the URL + ?action=ping into a browser tab. You should see {"ok":true,"message":"NAFL PTM Backend is live"...}

🔌 Part 2 — Connect the Portal to Your Backend

Step 1 — Open index.html in a text editor
Right-click → Open with → Notepad (Windows) / TextEdit (Mac) / any code editor.
Step 2 — Find this line (Ctrl+F / Cmd+F):
const API_URL = 'PASTE_YOUR_APPS_SCRIPT_URL_HERE';
Step 3 — Replace with your URL from Part 1, Step 6, then save:
const API_URL = 'https://script.google.com/macros/s/AKfycb..../exec';
✅ Done! The portal is now wired to the shared database.

🚀 Part 3 — Deploy to GitHub Pages

Step 1 — Go to github.com → Sign up → Verify email
Step 2 — + → New repository → Name: nafl-ptm → Public → Create
Step 3 — Click "uploading an existing file" → drag in your edited index.html → Commit changes
Step 4 — Settings → Pages → Source: Deploy from a branch → Branch: main / (root) → Save
Step 5 — Visit (wait ~2 min):
https://YOUR-GITHUB-USERNAME.github.io/nafl-ptm/
🎉 Fully live! Admin uploads on any device → every teacher sees it instantly (or taps 🔄 Refresh).

🔗 Shareable Class Links (for Circulars)

After generating the whole-school schedule, click 🔗 Get Class Links in the Admin panel (Step 5). This gives you a list like:

ClassLink Name
Grade 1, Sec AGrade1A_Timeslot
Grade 9, Sec AGrade9A_Timeslot

Click Copy next to any class, or 📋 Copy All to grab every link at once. Paste these directly into your circular — each link opens a clean page showing just that class's schedule, with a Download PDF / Excel / Print button and no login required.

⚠️ These links only work correctly after the portal is deployed at its final GitHub Pages URL — the link is built using whatever URL is currently in your browser's address bar, so always generate the links from the live site, not while testing locally.

🟪 Exporting for Microsoft Teams / Excel Online

The 🟪 Excel for Teams button (next to the other export buttons) downloads a completely standard .xlsx file — no special setup needed. To use it in Teams:

  1. Click 🟪 Excel for Teams to download the file
  2. Open your Microsoft Teams channel → go to the Files tab
  3. Click Upload and select the downloaded .xlsx file
  4. It opens directly in Excel Online inside Teams — no conversion needed

This is the same file format as the regular Excel export — Teams and Excel Desktop both read standard .xlsx natively.

🛠️ Updating the Backend Later

  1. Google Sheet → Extensions → Apps Script → edit code → Save
  2. Deploy → Manage deployments → ✏️ on existing deployment → New version → Deploy
  3. Same URL keeps working — no need to touch index.html again
